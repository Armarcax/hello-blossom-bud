# HAYQ Pine Script Strategy

This directory contains the HAYQ trading strategy for TradingView, written in Pine Script v5.

## Structure

```
pine-script/
├── hayq_strategy.pine    # Main strategy file
└── signals/              # Signal modules
    ├── ema_cross.pine    # EMA crossover signals
    └── rsi_levels.pine   # RSI overbought/oversold signals
```

## Main Strategy: `hayq_strategy.pine`

The HAYQ Modular Strategy combines multiple technical indicators to generate trading signals:

- **EMA Cross**: Fast/Slow EMA crossover detection
- **RSI Levels**: Overbought/Oversold conditions

### Parameters

- `Enable Alerts`: Toggle for alert notifications
- `Fast EMA Length`: Period for fast EMA (default: 12)
- `Slow EMA Length`: Period for slow EMA (default: 26)
- `RSI Length`: Period for RSI calculation (default: 14)
- `RSI Overbought`: Upper threshold (default: 70)
- `RSI Oversold`: Lower threshold (default: 30)

### Signal Logic

**Long Signal**: Triggered when:
- Fast EMA crosses above Slow EMA
- RSI is below overbought level

**Short Signal**: Triggered when:
- Fast EMA crosses below Slow EMA
- RSI is above oversold level

## Signal Modules

### EMA Crossover (`signals/ema_cross.pine`)

Implements moving average crossover strategy:
- `emaLong`: Fast EMA crosses above Slow EMA (bullish)
- `emaShort`: Fast EMA crosses below Slow EMA (bearish)

### RSI Levels (`signals/rsi_levels.pine`)

Implements RSI-based momentum signals:
- `rsiLong`: RSI is below overbought threshold
- `rsiShort`: RSI is above oversold threshold

## Usage in TradingView

1. Open TradingView and navigate to Pine Editor
2. Copy the contents of `hayq_strategy.pine`
3. Click "Add to Chart"
4. Configure parameters as needed
5. Enable alerts for automated notifications

## Integration with HAYQ Bot

The signals generated by this strategy can be integrated with the HAYQ Telegram bot through:

1. TradingView webhook alerts
2. Bot's `signals.py` module for processing
3. Automated trading execution via `trader.py`

## Customization

To add new signal modules:

1. Create a new `.pine` file in `signals/` directory
2. Define signal logic with `*Long` and `*Short` variables
3. Include the signal logic in main strategy
4. Combine with existing signals using AND/OR logic
